TCP/IP网络层次分析

Background

由于疫情原因，本学期的所有考试都改为提交相关的作业，虽然不用复习，这是做这种逻辑严密的论述对于我们还是有困难的，特别是计网这种硬核课程。以个人学习网络的经验，把这些层的多多少少的带一点出来，希望不要闹笑话。

OSI的五层协议

OSI的标准制定的是一种7层协议，但是因为太过于复杂，并不实用，这一篇文章我还是按照TCP/IP的五层协议来叙述。

经典的TCP/IP五层协议分为：

  • 应用层：离计算机程序最近的一层，主要为计算机程序提供网络接口，并且承担相关的编码、解码和会话管理的上层功能
  • 运输层：建立主机到主机的一个链接，为上层提供数据传输服务，屏蔽下层的数据通信细节
  • 网络层：基于IP地址来选择合适的路由，通过一个个路由器节点将分组传送到目的IP主机
  • 数据链路层：将上层的分组封装成帧，并将下层的比特流组装成帧，完成数据在链路上的透明传输
  • 物理层：规定比特流的传输，主要规定一些电气特性和机械特性

每一层次都有对应的设备，不同层次承担不同的责任，即使相同的层次在校园不同的区域之间的重要性也是不一样，下面分层次来逐个谈一谈。

应用层

应用层是我们在编写程序时直接使用的网络接口，在校园网中因为资源的私有性程度较高更多的会采用https协议，在学生提交作业的场景下又需要用到ftp协议，下面简单的谈一下这两个协议。

https：https是架设在http协议之上的超文本安全传输协议，全程Http on SSl（TLS），解决了http明文传输、无法提供服务器身份认证、保护账户和保持用户通信的缺点。https集成了加密、数据完成性和身份认证三大安全特性，能够有效防止信息监听和中间人攻击。

非对称加密：非对称加密的好处是加密和解密分别使用公钥和私钥，正是因为公钥和私钥具有很强的联系，所以非对称加密除了能够很好的加密我们的数据之外还能帮助我们做有效的身份认证，私钥加密的数据只能由公钥解密，反之亦然，缺点则是加密速度比较慢。

对称加密：对称加密秘钥和解密秘钥是相同的，加解密速度快，缺点的需要保持秘钥的安全性，一旦秘钥丢失安全性就不在

https很好的结合了这两种加密方式的优点，使用非对称加密（证书、信息摘要）完成身份认证，使用对称加密完成普通传输数据的加密和解密，达到了安全性和性能的平衡。

https的主要请求步骤：

  1. client发起请求
  2. server向client返回公钥
  3. client验证公钥的正确性，公钥一般是随着证书（CA）一起返回，client通过已经安装在操作系统中的可信赖的权威证书去验证返回的证书的真假
  4. client使用公钥加密一个对称秘钥的key发送到server
  5. server使用私钥解密得到对称秘钥key
  6. 之后二者通信的内容都是利用这个对称秘钥key进行高速加密

所以https很好的综合这两种加密方式的特点，用非对称加密验证身份，传输少量数据，保证安全性。然后利用对称加密进行正常数据传输的加密，保证加密解密的计算性能。

传输层

传输层作为我们网络协议中的重点，在校园网中一些科研数据的传输需要保证数据传输的可靠性和稳定性，这里我们着重讲一下TCP协议。说到TCP协议，绕不过去的就是流量控制和拥塞控制这两个话题。

流量控制：流量控制是为了解决发送方和接收方之间的数据处理速率不平衡的问题，目的是要让发送方的发送速率不要太快，要让接收方来得及接收。滑动窗口是这一措施中最经典的算法：

  • 发送方和接收各自维护一个窗口
  • 接收方在收到发送方的数据之后计算自己的剩余窗口大小，所以回给发送发的是信息不仅包括希望接收到的下一个开始字节，也包括一个最大可接收的字段
  • 所以接收方的接收速率（滑动窗口的大小）是整个数据传输的关键

以上情况会出现两个问题：

  1. 发送方的发送时机：加入接收方窗口足够，发送方从上游接收数据的速率为每秒1byte，那么如果收到1byte数据就加上头部发出去将会导致TCP连接的利用率大大降低，头部信息远远多于真正的数据信息，解决方法是使用Nagle算法优化发送时机：
    • 发送方把第一个字节的数据发送出去，剩余的缓存（相当于一个探测作用）
    • 收到前一个报文段的确认之后发送缓存的数据
    • 如果缓存数据达到发送窗口的一半或是最大报文段就发出去
  2. 糊涂窗口综合征：假如接收方上游读取数据较慢，同样一秒1byte，那么接收方会向发送方发送窗口剩余大小为1byte，同样会导致信道利用率不高，解决方法：
    • 接收方等待一定的时间，使得窗口可以容纳一个最大报文段再向发送方发送数据
    • 等待窗口空出一般向发送方发送信息

拥塞控制：拥塞控制是为了防止在网络状况不好的时候有过多的数据继续注入网络，使网络中的路由器或者是链路不过载，这是一个全局性的过程。TCP的拥塞控制主要有四个算法（发送方维持一个拥塞窗口c，如果网络好就增大窗口，网络比较慢就减小窗口这是核心思想）：

  • 慢开始：开始是c = 1，然后每一次发送报文如果接收方成功的收到则增大拥塞窗口，增大策略为：c = c * 2

  • 拥塞避免：c不能够按照指数一直增大，到达了相关的慢开始门限ssthresh之后，每次接收方的成功接收，增大策略变为c = c + 1

    如果网络出现超时，那么将c再次置位1，并开始慢开始算法，并且ssthresh = ssthresh / 2

  • 快重传：当接收方未收到一个报文段之后立即向发送方发送三次未收到信息，当发送方一连收到三个确认之后立即重传丢失的报文段

  • 快恢复：发送方收到三个丢失确认之后会判断此时网络并非一定就不好，毕竟还能收到三个确认，所以发送方将自己的ssthresh减小，减小策略为ssthresh = ssthresh / 2，然后重新开始慢开始算法

网络层

校园网是一个典型的局域网（LAN），网络层最主要的网络设备是路由器Router，路由器可以通过其内部存储的路由表和转发表确定路由信息，也就是我们常说的根据目的IP地址选择下一跳（下一个节点）路由器。路由器中的路由信息又分为静态路由和动态路由，由于静态路由需要人工配置，在校园网这样规模较大的平台不太实用，重点讨论动态路由的RIP协议和OSPF协议，看看这两种协议分别是以什么样的形式生成路由信息。另外路由器工作在网络层，也会出现网络拥塞的问题，一旦路由器的存储空间用完，后面进入路由器的分组就会丢弃，那么网络层是如何解决这个问题的呢？

RIP：RIP协议全称Routing Infomation Protocl，是一种基于距离向量的路由选择协议，每个路由器都要维护一个自己到其它网络的距离向量，从计算到达某一目的网络的最近距离。RIP协议具有以下特点：

  1. RIP协议中的Router只跟相邻的Router交换路由信息
  2. RIP协议只保存到某个路由器的最近距离向量
  3. 在RIP协议中，每个路由器并不知道整个网络拓扑的情况

下面通过一组路由器来模拟RIP算法，假设相邻路由器的数据交换在瞬间完成，路由器交换信息的时间间隔为t。初始网络拓扑中只有R0和R1路由器,二者距离为2，下面来模拟RIP过程：

  • 0：初始情况，[R0, (R1, 2)], [R1, (R0, 2)]

  • 1t：R2接入网络与R0和R1的距离分别是1和2，此时显然因为相邻是最短的，很容易得出路由信息，[R0, (R1, 2), (R2, 1)], [R1, (R0, 2), (R2, 2)], [R2, (R0, 1), (R1, 2)]

  • 2t：R3接入网络，与R1和R2的距离非别是2和4，此时R0就会接收来自R1和R2的信息，到达R3的最近距离分别是2和4，R0根据自己到R1（2）和R2（1）的距离计算得到到达R3的最近距离是2+2=4，其余路由同理，生成路由信息[R0, (R1, 2), (R2, 1), (R3, 4)], [R1, (R0, 2), (R2, 2), (R3, 2)], [R2, (R0, 1), (R1, 2), (R3, 4)]

后面的步骤就省略了，只是重复这个过程，所以你可以认为RIP这一个比较精明的协议，它每一条路都只求最短的那一条，绝对不做亏本生意。当然RIP只有工作在网络较好的情况下才能发挥优势，你想一下，一个人每次走只抄小路，那么只有在小路和大路交通状况相同时才有优势，可是现实就是小路一般是比较难走的。所以一旦RIP选择的路线出现了问题，那么网络拥塞的状况就很难解决，因为它只会这一条路，没办法合理的调度。

OSPF：OSPF全称Open Shortest Path First，是一种开放最短路径算法。之所以叫做最短路径算法是因为使用了Dijkstra提出的最短路径算法，但是OPSF协议中的路由器并非只知道最短路径，严格来说OSPF中的路由器知道到达另一个网络的所有路径。OSPF协议具有以下特点：

  1. 路由器与相邻路由器交换信息之后会将相邻路由器的信息继续抛出去
  2. 路由器只需要发送自己与周围的路由器的连接信息即可
  3. 只有在路由信息发生改变的时候才会使用洪泛法发送路由信息

先来理解一下洪泛法，路由器R1收到一个相邻路由器R2的路由信息之后会继续将这个路由信息发送到除R2之外的相邻路由器，想一想一个网络中的路由器Rn，经过一次又一次的转发，Rn最后会收到[1, n - 1]个路由器的信息，那么Rn完全可以通过这个路由信息构建整个网络的拓扑图，对于网络中的另外n - 1个路由器来说亦是如此。因此在OSPF协议中每个路由器都能知道整个网络的拓扑。下面模拟OSPF算法，初始只有R0和R1两个路由器，距离为2：

  • 0：网络拓扑[(R0, R1, 2)]
  • 1t：R2接入网络与R0和R1的距离分别是1和2，洪泛法更新之后的网络拓扑[(R0, R1, 2), (R0, R2, 1), (R1, R2, 1)]
  • 2t：R3接入网络，与R1和R2的距离非别是2和4，洪泛法更新之后网络拓扑[(R0, R1, 2), (R0, R2, 1), (R1, R2, 1), (R1, R3, 2), (R2, R3, 4)]

加入现在R0要发一个信息到R3，可以找到4条路径：

  • R0->R1->R3， 4
  • R0->R2->R3， 5
  • R0->R1->R2->R3， 6
  • R0->R2->R1->R3， 5

至于怎么算的可以去看无向图中的DFS和BFS算法，如果你想求最短路径那就用Dijkstra算法。相比于RIP的极端，OSPF明显是个顾全大局的协议，路由器可以知道去往每一个网络的路径，如果近路好走就走近路，不好走绕远一点也能很好的到达，毕竟在OSPF这里条条大路通罗马。在一条路出现拥塞的时候，OSPF可以轻松地换一条路线继续前进。

当然说了这么多并不是说OSPF要优于RIP，因为在网络较好的情况下或者某一个区域之内我们肯定会优先选择RIP协议，在比较复杂的时候可以通过OSPF更加合理的规划。另外OSPF因为需要交换的信息太多，这些洪泛的数据包本身就会占据一定的带宽和相关的计算性能，所以校园内在选择这两种协议的时候也要仔细考虑。

RED：RED全称Random Early Detection，也就是随机早期探测算法。路由器转发分组遵循先进先出的原则，一旦路由器缓存满了，后面的所有分组都将被丢弃，这样连续的丢包一般是不能忍受的，我们可以提前做一些随机丢包来避免：

  • 路由器缓存空间[1, n]，取一个1 < x < n
  • 当数据大小小于x时说明空间足够，正常接收
  • 当数据大小大于x时说明空间已经紧张了，接收到一个分组之后按照1 / (n - x)的概率丢弃

这样我们就将可能会丢的包分散开了，避免了一个请求大量丢包的状况。

数据链路层

数据链路层主要是完成链路上帧的传输，封装上层分组为帧，收取下层比特流并组合为帧进而转为分组传递给上一层，为网络层提供透明无感知的数据传输服务，相对于前几层，数据链路层的算法实际上比较少。对于链路层，主要的关注点是如何完成帧的封装、透明传输和差错校验，下面重点谈一下这三个方面。

帧的封装：在一段数据前后添加相关的首部和尾部，将帧与帧之间区分开来，接收方皆可以实现一帧一帧的接收，不会混乱。这样有了确定的首部和尾部，也可以很好的检测帧的完整性。

透明传输：透明传输是指无论是什么数据，链路都应该无差别的传输，链路不管数据的具体内容，但是因为界定符的原因，导致如果数据中出现界定符，那么会导致帧的边界检测失败，这是需要改进的，主要方法插入转义字符，类似于C语言中特殊ASCII码点的转义，例如\r, \n等等

差错校验：链路层的差错校验重点关注的是校验，并不会对差错做出过多的纠正工作。在这一方面使用的广泛的算法是CRC循环冗余校验算法。CRC的具体步骤是：

  • 数据划分为组，每组K个比特
  • 选择合适的除数
  • 看选定除数的二进制位数，然后再要发送的数据帧上面加上这个位数-1位的0，然后用新生成的帧以模2除法的方式除上面的除数，得到的余数就是该帧的CRC校验码。注意，余数的位数一定只比除数位数少一位，也就是CRC校验码位数比除数位数少一位，如果前面位是0也不能省略。
  • 将计算出来的CRC校验码附加在原数据帧后面，构建成一个新的数据帧进行发送；最后接收端以模2除法方式除以前面选择的余数，如果没有余数，则说明数据帧在传输过程中没有出错。

物理层

物理层主要是关注网络设备的电气特性和机械特性，保证比特信号的正常传递和识别。
